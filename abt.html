[ Brudl ]

Brudl is a flavor of HDL. It helps in designing electrical components. It has a pretty simple syntax which doesn't get in your way when you want to be efficient.

A sample brudl file would look like this.


LOAD [
   filename_with_path
]

#nand
SIM
IN ib1[2]
OUT ob1
CON
   t1 = and(ib1[0], ib2[1])
   ob1 = not(t1)
END

#mux
IN sp1 ib1 ib2
OUT ob1
CON
   t1 = not(sp1)
   t2 = and(sp1, ib2)
   t3 = and(t1, ib1)
   ob1 = or(t2, t3)
END

HDL Description :

LOAD
Like C's #include, it will load the whole of the file whose name is given and copy its contents over to the current file before interprettation occurs.

#
Declares the name of the chip.

IN
Declares the inputs that a component will accept.

OUT
Declares the outputs that a component will accept.

CON
Denotes the start point for where the component's working is described.

END
Denotes the end of the component's description.

not
One of the three standard gates the interpretter understands by default.

and
One of the three standard gates the interpretter understands by default.

or
One of the three standard gates the interpretter understands by default.

SIM
A flag that denotes which chip is marked for simulation. Only one out of all the chips in the file (including those that are in the files that are loaded) can be marked for simulation.
